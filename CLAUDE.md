# 话好眠 — 睡眠健康微信小程序

## 项目概览

**话好眠**是一款基于微信云开发的睡眠健康监测小程序，提供每日睡眠报告、长期趋势分析、睡眠建议与用户中心功能。

- AppID: `wx399b0d556538592e`
- 小程序根目录: `miniprogram/`
- 云函数根目录: `cloudfunctions/`
- 微信基础库版本: 2.20.1

## 页面结构

| 路径 | Tab 标题 | 功能 |
|---|---|---|
| `pages/daily/index` | 每日 | 当日睡眠详情报告（睡眠分期、呼吸、心率、体动、声音） |
| `pages/longterm/index` | 长期 | 多维度睡眠趋势折线图 |
| `pages/advice/index` | 建议 | 睡眠风险评估与改善建议 |
| `pages/my/index` | 我的 | 用户中心 |
| `pages/temple/index` | — | 导航辅助页（从每日页跳转） |

## 技术要点

### 图表渲染
所有图表使用 **微信 Canvas 2D API** 原生绘制，无第三方图表库：
- `wx.createSelectorQuery()` → `.fields({ node: true, size: true })` 获取 canvas node
- 必须在 `onReady()` 之后才能绘制（DOM 已就绪）
- 需支持 DPR 缩放：`canvas.width = width * dpr; ctx.scale(dpr, dpr)`
- Toggle 显示/隐藏面板后需在回调中重新调用对应 draw 函数

### 云函数
`cloudfunctions/quickstartFunctions/index.js` 使用 `wx-server-sdk`，以 `event.type` 字段作为路由分发：
- `getOpenId` / `getMiniProgramCode` / `createCollection`
- `selectRecord` / `updateRecord` / `insertRecord` / `deleteRecord`（操作 `sales` 集合，示例数据）

### 数据状态
当前页面数据**全部为硬编码 mock 数据**，尚未接入真实设备或云数据库。

## 算法需求生成工作流（强制）

当用户在对话中要求生成任何算法/规则/判定逻辑时，**必须**严格遵循以下流程，无需用户每次重复说明：

### 1. 算法格式四要素（缺一不可）

每个算法必须包含：

- **输入**：字段名 + 类型 + 含义 + 单位/来源（表格形式）
- **算法本体**：阈值表 / 判定规则 / 公式 / 伪代码，**任选最简洁的一种**
- **输出**：字段名 + 类型 + 示例
- **简洁明了**：不堆砌冗余说明，能用表格不用段落

### 2. 必同步写入算法文件

算法**不只**在对话中给出，**必须**同步写入对应页面的算法文件：

| 页面 | 算法文件 |
|---|---|
| 每日 tab | `algorithms_daily.md` |
| 长期 tab | `algorithms_longterm.md` |
| 我的 tab | `algorithms_mine.md` |

### 3. 板块顺序对齐

在算法文件中按页面板块顺序排列章节，新算法**自动归类**到对应板块：

| 页面 | 板块顺序（章节号） |
|---|---|
| 每日 | §1 心脏健康总览 → §2 今日打卡 → §3-§5 心脏具体数据 → §6 呼吸健康 → §7 睡眠健康 → §8 声音监测 → §9 折叠态状态徽章 |
| 长期 | §1 综合诊断 → §2 心脏年龄 → §3-§5 心脏长期 → §6-§7 呼吸长期 → §8 睡眠长期 → §9 声音长期 → §10 风险卡 → §11 长期建议 |
| 我的 | §1 设备 → §2 被监测人 → §3 监测参数 → §4 预警 → §5 物联卡 → §6 分享 → §7 会员 → §8 通用设置 |

### 4. 自动分类规则

无需用户指明算法归属，根据描述自动判断：
- 含「心率/HRV/ECG/心脏」→ 心脏板块
- 含「呼吸/AHI/暂停」→ 呼吸板块
- 含「睡眠/分期/翻身/体动/恢复」→ 睡眠板块
- 含「鼾声/梦话/咳嗽/噪音」→ 声音板块
- 含「设备/绑定/固件」→ 我的页 §1
- 含「baseline/预警/参数」→ 我的页 §3-§4
- 含「会员/试用/付费」→ 我的页 §7

### 5. 对话同步表格

写入文件后，对话中**必须**以 markdown 表格形式呈现速览版，包含：
- 输入表
- 算法核心表（阈值/规则）
- 输出/示例
- 实现位置

### 6. 完成进度维护

每次新增算法时，更新对应文件章节顶部的「完成进度清单」，把 `[ ]` 改为 `[x]`。
