## §3 监测参数自动推算

> 关键参数（夜间心率参考范围 baseline、报告时段、离床时间等）的自动推算 + 手动覆盖。

### §3.1 夜间心率 baseline 自动推算

待写入：
- [ ] 输入：用户年龄 + 最近 7 天静息心率均值
- [ ] 公式（候选）：`baselineMin = restingHR - 5`，`baselineMax = restingHR + 8`
- [ ] 冷启动：新用户首晚的默认值（按年龄段：18-35→55-72；35-55→58-75；>55→60-78）
- [ ] 用户手动设置后不再自动覆盖

### §3.2 报告时段自动推算

待写入：
- [ ] 输入：最近 7 天的入睡/起床时间分布
- [ ] 输出：建议的"开始监测时段" / "结束监测时段"
- [ ] 异常值处理（如熬夜通宵不计入）

### §3.3 离床时间阈值

待写入：
- [ ] 离床多久判定为"起床"（默认 ≥30 分钟无信号）
- [ ] 上厕所等短暂离床的过滤

### §3.4 心率统计时长

待写入：
- [ ] "静息心率"取整夜最低 HR 的持续时段（默认 ≥5 分钟）
- [ ] 是否允许用户自定义统计窗口

## §4 预警触发与通知

> 3 种预警类型：心率预警、呼吸频率预警、离床预警。

### §4.1 触发规则（统一三段式）

| 字段 | 含义 |
|---|---|
| `enabled` | 是否启用 |
| `effectivePeriod` | 生效时段：`{start: "23:00", end: "07:00"}` |
| `thresholdHigh` | 高阈值 |
| `thresholdLow` | 低阈值 |
| `durationSec` | 超阈值持续时间（避免瞬时抖动） |
| `notifyChannels` | 通知渠道：`["wechat", "sms", "phone"]` |

### §4.2 各预警的默认值

待写入：
- [ ] 心率预警默认（高：120 / 低：40 / 持续 30s）
- [ ] 呼吸频率预警默认（高：25 / 低：8 / 持续 60s）
- [ ] 离床预警默认（夜间离床 >30 分钟）

### §4.3 通知频次与去重

待写入：
- [ ] 同一晚同一预警最多通知 N 次
- [ ] 优先级（电话 > 短信 > 微信）
- [ ] 用户回执后停止当晚后续通知

## §7 会员状态判定

### §7.1 状态字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `isMember` | bool | 是否会员（含试用） |
| `isTrial` | bool | 是否试用中 |
| `trialStartedAt` | datetime | 试用开始时间 |
| `trialEndsAt` | datetime | 试用结束时间 |
| `planType` | enum | `trial`/`monthly`/`yearly`/`expired` |
| `expiresAt` | datetime | 当前套餐到期 |
| `autoRenew` | bool | 是否自动续费 |
| `postTrialPlan` | enum | 试用结束后默认套餐 |

### §7.2 状态判定规则

待写入：
- [ ] 试用期：从 `trialStartedAt` 起 7 天内 `isMember = true`，到期自动转 `expired` 或 `postTrialPlan`

- [ ] 过期后立即降级 vs 宽限期
- [ ] 退款/取消订阅的状态流转

### §7.3 功能门控
