### 3.3 心率数据卡（`hrChart` 对象）

| 前端字段 | 后端字段 | 类型 | 说明 |
|---|---|---|---|
| 心率范围显示 | `hrChart.rangeDisplay` | string | "55-69" 形式（min-max，整数） |
| 折线图数据点 | `hrChart.series` | `int[]` | **定长 96 点**，每点为一段时间窗的平均 HR。若整夜不足 96 段则后端填 `null`，前端做断点处理 |
| Y 轴范围 | `hrChart.yAxis` | `{min, max}` | 默认 `{50, 80}`，可由后端按当夜数据动态调整（向外扩 5 bpm） |
| 时段轴标签 | `hrChart.timeTicks` | `string[]` | `["01:00", "03:00", "05:00"]` 三档；按入睡-起床均分 |
| 参考范围 baseline | `hrChart.baseline` | `{min, max}` | 用户在「监测参数设置」配置的夜间心率参考范围，例如 `{55, 65}` |
| 状态徽章 | `hrChart.status` | enum | `normal`/`warning`/`risk` |
| 文字解读 | `hrChart.analysis` | object | 见 §5.1 |

> **96 点的由来**：整夜 8 小时 ÷ 5 分钟一段 = 96 段，与既有趋势图惯例一致。后端按用户实际入睡-起床时长重采样。

### 3.4 心动图卡（`ecg` 对象）

| 前端字段 | 后端字段 | 类型 | 说明 |
|---|---|---|---|
| ECG 波形 | `ecg.series` | `float[]` | **定长 48 点**，归一化到 0-1（前端按 Y 轴映射） |
| 片段时长 | `ecg.durationSec` | int | `3`，固定 3 秒片段 |
| 片段时间戳 | `ecg.sampleAt` | string | 该片段对应的真实时刻（如 `"03:14:22"`），用于跳转长期页定位 |
| 状态徽章 | `ecg.status` | enum | `normal`/`warning`/`risk` |
| 检出异常事件 | `ecg.events` | `string[]` | `["early_beat", "missed_beat", "st_elevation", ...]`，无异常返回 `[]` |
| 文字解读 | `ecg.analysis` | object | 见 §5.2 |

> **片段选择策略（建议）**：从整夜 ECG 中选 1 个 "典型片段"，即心率最稳、无异常的 3s 片段（常在深睡阶段）。若有异常则**优先返回异常片段** + `status = warning`。

## 4. 算法 / 规则规范

### 4.1 心脏健康指数 (0-100) 计算

**推荐公式（待医学顾问审核）**：

```
heartIndex = 100
  - HRV_penalty       (0-25 分)
  - restingHR_penalty (0-20 分)
  - AHI_penalty       (0-15 分)   // 跨模块：从呼吸健康取
  - rhythm_penalty    (0-25 分)   // 节律稳定性
  - ECG_event_penalty (0-15 分)   // 检出的 ECG 异常事件
```

**各惩罚项规则**（建议初始值，可调）：

| 惩罚项 | 测量字段 | 0 分阈值 | 满分阈值 | 计算 |
|---|---|---|---|---|
| HRV 惩罚 | SDNN (ms) | ≥50 (健康) | <20 (高危) | 线性插值 |
| 静息心率惩罚 | 整夜最低 HR (bpm) | <60 (健康) | >75 (偏高) | 线性插值 |
| AHI 惩罚 | 呼吸暂停指数 (次/h) | <5 (正常) | >30 (中重度) | 阶梯：<5→0；5-15→5；15-30→10；>30→15 |
| 节律惩罚 | RR 间期变异系数 (%) | <5% | >12% | 线性插值 |
| ECG 事件惩罚 | events 数组 | `[]` | 出现早搏/漏搏/ST 等 | 每项 +3 分，最高 15 |

**等级判定**：

| value | level | levelLabel |
|---|---|---|
| ≥85 | `excellent` | "优秀" |
| 60-84 | `normal` | "正常" |
| <60 | `needsAttention` | "需要关注" |

> ❓ **待决策**：权重 25/20/15/25/15 是否合理？阈值是否需要按年龄/性别分层？建议医学顾问出 v1.0 规则。

### 4.2 4 项核心指标的等级文案规则

| 指标 | 健康区间 | 文案规则 |
|---|---|---|
| 心率范围 | min ≥50 且 max ≤80 | 区间宽 ≤15: "夜间平稳"；区间宽 16-25: "波动正常"；>25: "波动较大" |
| HRV (SDNN) | ≥50 ms | ≥50: "状态优秀"；30-49: "恢复中等偏好"；20-29: "偏低需关注"；<20: "显著偏低" |
| 平均心率 | 45-70 bpm | 文案固定显示参考区间："参考45-70bpm"，前端不变文案，后端可在 `raw` 字段返回是否超出 |
| rMSSD | 600-1200 ms（待医学确认范围） | 文案固定参考区间："参考600-1200ms" |

> ❓ **待决策**：rMSSD 单位与参考区间。临床常见 rMSSD 范围是 15-50 ms（成人），而代码 mock 是 892 ms，疑似 RR 间期均值或单位口径不同，**需与厂家明确口径**。

### 4.3 折叠态状态徽章状态机

| 字段 | 判定逻辑 | 输出 |
|---|---|---|
| 夜间心率 | 整夜 HR 均值在 baseline 内 & 无突发 >100 bpm | `normal` / "未见异常" |
| 夜间心率 | 整夜均值超过 baseline ±10 bpm 或 HRV <20 ms | `warning` / "需关注" |
| 夜间心率 | 出现 >120 bpm 持续 ≥5 分钟 | `risk` / "异常" |
| 心动图 | `ecg.events` 为空 | `normal` / "未见异常" |
| 心动图 | 仅检出"偶发早搏"等低级别异常 | `warning` / "偶有早搏" |
| 心动图 | 检出室速/ST 抬高等高危事件 | `risk` / "需复诊" |

> 严重程度向上传递：主卡折叠态显示 2 个徽章中较严重的一个的颜色（用于副标题区域整体染色，可选）。

## 5. 文字解读生成规则

### 5.1 心率文字解读（`hrChart.analysis`）

**数据结构**：

```json
"analysis": {
  "headline": {
    "template": "整夜心率维持在 {{range}} 次/分，与你设定的「夜间心率参考范围」{{baseline}} {{matchVerb}}，整体表现{{tone}}。",
    "vars": {
      "range": "55-69",
      "baseline": "55-65",
      "matchVerb": "基本吻合",
      "tone": "良好"
    }
  },
  "bullets": [
    {
      "text": "曲线整体平滑，未见骤升骤降，提示夜间「{{term:交感神经}}」活跃度低，心脏处于充分休息状态。",
      "vars": {}
    },
    {
      "text": "凌晨 5 点后出现 ~{{rebound}}bpm 轻度回升、短暂超出你的范围上限 {{exceed}}bpm，属正常觉醒前生理反应，无需关注。",
      "vars": { "rebound": 8, "exceed": 4 }
    }
  ]
}
```

**前端渲染规则**：
- `{{var}}` 直接替换为数字 / 字符串，并以 `<text class="bold">` 加粗
- `{{baseline}}` 渲染为 `<text class="baseline-pill">` 胶囊样式
- `{{term:xxx}}` 渲染为 `<text class="tooltip-term" data-term="xxx">`，点击触发 §6 tooltip

**模板池（建议规则化产出）**：
- **headline matchVerb**：均值差 ≤3 bpm → "基本吻合"；4-8 → "略有偏移"；>8 → "明显偏离"
- **headline tone**：min/max 全在 baseline 内且 HRV≥30 → "良好"；其余 → "中等" / "需改善"
- **bullets 数量**：固定 2-3 条，按"曲线特征 + 异常时段 + 与睡眠分期关联"3 个角度生成

### 5.2 心动图文字解读（`ecg.analysis`）

**数据结构同 §5.1**，模板池建议：

| 触发条件 | headline 模板 |
|---|---|
| events 为空 | "心电波形形态规整，「{{term:R波}}」清晰锐利，节律间隔均匀。" |
| 仅有偶发早搏 | "心电波形整体规整，检出 {{n}} 次{{event}}，{{severity}}。" |
| 高危事件 | "检测到{{event}}，建议结合症状到心内科复诊。" |

| 触发条件 | bullets |
|---|---|
| 正常 | ① "P-QRS-T 各波段比例协调，反映{{term:窦性心律}}稳定…" ② "未见早搏、漏搏、ST 段抬高等异常信号，{{term:rr间期}}变异在正常范围。" |
| 异常 | 具体描述异常类型 + 关联风险 + 建议 |

### 5.3 后端实现建议

- 模板表存数据库 / 配置中心，**key = 触发条件**，**value = 模板字符串**
- 算法服务输入：当夜全部指标 + ECG events → 输出：选中的模板 + 变量值
- 前端只需做字符串替换 + 术语 tooltip 渲染，不参与判断
- **不要让前端做判断**，否则医学规则散落两端难以维护

```json
  "rr间期": "两次心跳之间的时间间隔，反映心率规律性"
}
```

> 后端缓存策略：长期不变，前端启动时拉取一次缓存到 globalData，避免每次解读都请求。

## 7. 推荐接口设计

### 7.1 主接口：`POST /daily/heart`

**请求**：
```json
{
  "openId": "oXXXX",
  "date": "2026-05-30",      // YYYY-MM-DD
  "deviceId": "X2-20240115001"
}
```

**响应（200）**：
```json
{
  "code": 0,
  "data": {
    "date": "2026-05-30",
    "summary": {
      "heartRate": { "status": "normal", "label": "未见异常" },
      "ecg":       { "status": "normal", "label": "未见异常" }
    },
    "index": {
      "value": 78,
      "level": "normal",
      "levelLabel": "正常"
    },
    "coreMetrics": [ /* §3.2 */ ],
    "hrChart": {
      "rangeDisplay": "55-69",
      "series": [60, 61, /* ... 96 项 */],
      "yAxis": { "min": 50, "max": 80 },
      "timeTicks": ["01:00", "03:00", "05:00"],
      "baseline": { "min": 55, "max": 65 },
      "status": "normal",
      "analysis": { /* §5.1 */ }
    },
    "ecg": {
      "series": [0.50, 0.46, /* ... 48 项 */],
      "durationSec": 3,
      "sampleAt": "03:14:22",
      "status": "normal",
      "events": [],
      "analysis": { /* §5.2 */ }
    }
  }
}
```

**异常响应**：

| code | 含义 | 前端处理 |
|---|---|---|
| 0 | 成功 | 正常渲染 |
| 4040 | 当晚无监测数据 | 整个板块灰显，文案："昨夜未监测到数据" |
| 4041 | 当晚数据不完整（如不足 2 小时） | 显示已有部分，缺数据的字段 = null，前端兜底 "数据不足" |
| 5001 | 厂家接口异常 | toast 重试 |

### 7.2 厂家接口适配层

**职责**：我方后端 = 厂家原始数据 → 我方业务结构 的转换层。

```
[厂家 API/SDK]
     │  原始指标(HRV/HR序列/ECG波形/...)
     ▼
[我方"数据适配服务"]   ← 字段映射、单位统一、降采样到 96/48 点
     │
     ├──► [心脏健康指数计算服务]    ← §4.1
     ├──► [指标等级文案服务]        ← §4.2/§4.3
     ├──► [文字解读模板服务]        ← §5
     ▼
[/daily/heart 接口聚合返回]
```

> 建议把 §4 / §5 实现为**独立的纯函数式服务**（无状态、易测试、可单测），与厂家适配层解耦，未来换设备只换适配层。

## 8. 数据存储建议

### 8.1 表结构（云数据库集合）

| 集合 | 字段 | 用途 |
|---|---|---|
| `heart_daily` | `openId, date, deviceId, hrSeries, hrAvg, hrMin, hrMax, hrv, rmssd, ecgClipSeries, ecgClipSampleAt, ecgEvents, index, indexLevel, createdAt` | 每用户每日一行，是上面接口的主数据源 |
| `heart_raw_ecg` | `openId, date, deviceId, rawSeries (压缩), createdAt` | 整夜原始 ECG（如有），仅长期存档，前端不直读 |
| `medical_terms_dict` | `term, definition, updatedAt` | §6 术语词典 |
| `heart_index_config` | `weights, thresholds, version, effectiveAt` | §4.1 算法权重，可配置 |

### 8.2 数据写入时机

1. 用户起床后（或定时 09:00）触发厂家数据拉取
2. 适配服务计算所有字段
3. 写入 `heart_daily`
4. 用户打开"每日"tab 时直接读 `heart_daily`，不再实时算
